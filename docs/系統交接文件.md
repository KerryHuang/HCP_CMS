## 1. 系統概述

HCP CMS（客服管理系統）是一套 **Windows 桌面應用程式**，整合信件收信、案件追蹤、知識庫管理、Mantis 同步、報表產生、Patch 整理等功能，供 HCP 客服團隊日常使用。

**核心目標：**
- 統一管理來自 IMAP / Exchange 的客戶信件
- 自動分類、建案、串接對話串
- 提供知識庫（KMS）供客服人員查詢解決方案
- 每月 Patch 整理自動化（含 Claude AI 補充說明）
- 產出追蹤表、月報 Excel

**使用者：** 3 人客服團隊（含 1 名技術負責人）

## 2. 整體架構

### 2.1 六層架構

```
┌─────────────────────────────────────────────────────┐
│  UI 層 (PySide6)                                     │
│  View / Widget / Dialog                              │
│  ↕ Signal / Slot                                     │
├─────────────────────────────────────────────────────┤
│  Core 層 (業務邏輯)                                   │
│  XxxManager / XxxEngine                              │
├─────────────────────────────────────────────────────┤
│  Services 層 (外部整合)                               │
│  MailProvider ABC / MantisClient ABC                 │
├─────────────────────────────────────────────────────┤
│  Scheduler 層 (背景排程)                              │
│  threading.Timer 遞迴排程                             │
├─────────────────────────────────────────────────────┤
│  Data 層 (Repository + FTS5)                         │
│  XxxRepository / FTSManager                          │
├─────────────────────────────────────────────────────┤
│  SQLite (WAL 模式 + FTS5 虛擬表)                      │
└─────────────────────────────────────────────────────┘
```

**原則：**
- 只能從上層呼叫下層，禁止跨層存取
- UI 層透過 Signal/Slot 與 Core 層溝通
- 所有 SQL 操作封裝在 Repository，Core 層禁止直接執行 SQL
- 全程共用同一條 `sqlite3.Connection`

### 2.2 依賴注入

`app.py::main()` 建立 `DatabaseManager`，取得單一 `conn`，逐層向下傳遞：

```
app.main()
  └─ DatabaseManager(db_path).initialize()
       └─ returns conn
  └─ MainWindow(conn, db_dir)
       └─ PatchView(conn) → MonthlyPatchTab(conn) → MonthlyPatchEngine(conn) → PatchRepository(conn)
       └─ CaseView(conn)  → CaseManager(conn)      → CaseRepository(conn)
       └─ ... (所有層共用同一條連線)
```

### 5.2 FTS5 虛擬表

- `cases_fts` — 案件全文索引（主旨 + 本文）
- `qa_fts` — 知識庫全文索引（問題 + 回覆）

搜尋管線：`jieba 斷詞 → 同義詞擴展 → FTS5 MATCH`，**禁止直接 LIKE 查詢**。

### 5.3 Schema 版本管理

- `DatabaseManager._apply_pending_migrations()` — 冪等 `ALTER TABLE` 遷移
- 每次修改 schema 後在 `database.py` 新增遷移腳本

### 5.4 `mantis_detail` JSON 欄位說明

`cs_patch_issues.mantis_detail` 欄位儲存豐富的 Mantis 資料：

```json
{
  "form_files": ["HRWF304"],
  "archive_name": "01.IP_11G.7z",
  "supplement": {
    "修改原因": "...",
    "原問題": "...",
    "範例說明": "...",
    "修正後": "...",
    "注意事項": "..."
  },
  "supplement_edited": true
}
```

`supplement_edited: true` 表示人工編輯過，AI 重新分析全部時會跳過。

## 7. 核心業務流程

### 7.1 信件處理管線（7 步，禁止跳步）

```
① 重複檢查（processed_files 比對 file_hash / message_id）
② 廣播信過濾（寄件人黑名單 / 主旨黑名單）
③ 欄位解析（寄件人、主旨、本文、日期；草稿 .msg 補回寄件人）
④ 自動分類（Classifier 依 6 種規則判斷產品/類型/優先級/技術人員）
⑤ 對話串比對（ThreadTracker 依主旨前綴 / message_id / in_reply_to 串接）
⑥ 建案/更新（新案建立 or 後續來信重開父案件）
⑦ QA 抽取（偵測 Outlook 引用格式，自動儲存待審核 KMS 條目）
```

### 7.2 每月大 PATCH 流程（9 步）

```
① 掃描資料夾 → PatchRepository.insert_patch()
② 匯入 Excel  → MonthlyPatchEngine.import_release_note()
③ 編輯 Issue  → UI 直接編輯（PatchRepository.update_issue()）
④ 簡轉繁     → MonthlyPatchEngine.convert_s2t()
⑤ 產生 Excel  → ExcelExporter.export_patch()
⑥ 驗證超連結  → MonthlyPatchEngine.verify_links()
⑦ 補充說明   → MonthlyPatchEngine.fetch_supplements()
⑧ 人工審核   → SupplementEditorWidget（補充說明 Tab）
⑨ 完成       → MonthlyPatchEngine.finalize()
```

### 7.3 KMS 搜尋管線

```
使用者輸入
  → jieba.cut()（中文斷詞）
  → SynonymRepository.expand()（同義詞擴展）
  → FTS5 MATCH（全文搜尋）
  → 結果排序（BM25 加權）
```

## 8. 跨層資料流

### 8.1 案件建立

```
EmailView._on_fetch_clicked()
  → CaseManager(conn).process_mail_file(path)
      → Classifier.classify(subject, body)
      → ThreadTracker.find_parent(subject, message_id)
      → CaseRepository.insert(case)
      → FTSManager.update_index(case_id)
      → KMSEngine.extract_qa(body)  # 第 7 步 QA 抽取
```

### 8.2 補充說明 AI 分析

```
MonthlyPatchTab._on_supplement_clicked()
  → Signal: open_supplement_editor.emit(issues)
  → PatchView._on_open_supplement_editor()
  → SupplementEditorWidget.load_issues(issues)

後台執行：
  threading.Thread → MonthlyPatchEngine.fetch_supplements(patch_id)
    → MantisSoapClient.get_issue(issue_no)    # 取 Mantis 原始資料
    → ClaudeContentService.extract_supplement()  # 呼叫 Claude API
    → PatchRepository.update_issue_supplement()  # 寫入 DB
  → Signal: _supplement_done.emit()  # 主線程安全更新 UI
```

### 8.3 執行緒安全原則

- 所有背景執行緒必須透過 **Signal** 傳回主線程，禁止直接操作 UI
- 新增背景 Signal 範例（已實作）：
  ```python
  _reanalyze_single_done = Signal(int, object)  # 單筆重分析完成
  ```
