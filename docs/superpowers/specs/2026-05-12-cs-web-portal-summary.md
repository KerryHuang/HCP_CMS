# 客服 Web Portal 簡介（5 分鐘版）

> 完整設計請看 `2026-05-12-cs-web-portal-design.md`
> 本文件給非技術讀者快速理解用

## 一句話

讓 YOGA、Rebecca 用瀏覽器自己更新案件狀態、進度、處理人員，不再透過 Jill 代操。

## 為什麼要做

目前情況：
- Jill 管 3 人客服團隊
- 所有案件更新只有 Jill 桌面 App 能做
- YOGA / Rebecca 要改個狀態，得請 Jill 動手 → Jill 變瓶頸

希望變成：
- YOGA / Rebecca 自己打開瀏覽器 → 看到自己負責的案件 → 自己改
- Jill 桌面 App 還是用，但專注做高級功能（報表、Patch、KMS）

## 客服 B/C 會看到什麼

打開瀏覽器，輸入 `http://你電腦IP:8080`：

1. **登入頁**：看到 3 個名字按鈕（jill / YOGA / Rebecca），點自己名字 → cookie 記住下次自動登入
2. **我的案件清單**：看到屬於自己的案件，包含：
   - 自己被指派處理 (`handler`) 的案件
   - 自己負責公司的案件（透過 `cs_staff_id`）
3. **點任一案件 → 編輯頁**：
   - 可改：狀態、進度、處理人員、優先度、技術負責人
   - 可加：補充紀錄、Mantis 連結
   - 看到：原信件主旨、寄件人、時間、補充紀錄歷史
4. **改完按存** → 完成

## 你（Jill）會看到什麼

跟客服一樣，但多一個：

5. **稽核 log 頁**（只有你能看）：誰、何時、改了哪個案件的哪個欄位

桌面 App 照舊用，沒有任何改變。

## 怎麼運作的

```
你的 PC（你開機時）
├── 桌面 App ──┐
├── 信件收取    │  都讀寫同一個 cs_tracker.db
├── Web Server ┘
└── DB 檔案

3 人瀏覽器 ──→ 連你 PC 的 8080 port
```

- 客服在公司辦公室：用 LAN 直接連你 PC
- 客服在家：裝 Tailscale（免費）→ 連你 PC

## 取捨：你要接受什麼

1. **你關電腦時，客服 B/C 不能用** —— 但本來就只在你開機時有新案件進來，不算新增限制
2. **沒有密碼，靠互信** —— 3 人團隊，瀏覽器點自己名字就進去，依賴 LAN/Tailscale 封閉網路保護
3. **LAN 內走 HTTP（不加密）** —— 內網無中間人風險，未來可改 Tailscale 內建 HTTPS

## MVP 砍掉的功能（之後 Phase 2 再做）

為了 6 天上線，這些先不做：

- 在 Web 新增整筆案件（電話客服 / 面談紀錄）
- 改分類欄位（產品 / 問題類型 / 錯誤類型）
- 改問題 / 原因 / 解法 3 欄
- 自訂欄位
- 在 Web 直接寄回信
- 移除 Mantis 連結
- 「孤兒案件」認領頁
- 桌面 App 加研發人員管理 tab

這些**桌面 App 上你照舊能做**，只是 Web 沒做。
