## 1. 安裝與初始設定

### 1.1 系統需求

- Windows 10/11 或 macOS 12+
- Python 3.10 以上
- 磁碟空間 200MB 以上

### 1.2 安裝步驟

**方式 A：從原始碼安裝（開發人員）**

```bash
git clone https://github.com/KerryHuang/HCP_CMS.git
cd HCP_CMS
pip install -e ".[dev]"
```

**方式 B：使用打包好的執行檔**

1. 從 Release 頁面下載 `HCP_CMS.exe`（Windows）或 `HCP_CMS.app`（macOS）
2. 放到目標資料夾
3. 雙擊執行

### 1.3 首次啟動

首次啟動時，系統會自動：
- 在 `%APPDATA%/HCP_CMS/`（Windows）或 `~/Library/Application Support/HCP_CMS/`（macOS）建立資料庫
- 建立空的 `cs_tracker.db`
- 載入繁體中文語系

建議首次啟動後先進行以下設定：
1. 前往 **系統設定** → 輸入客服人員姓名
2. 設定信件來源（IMAP / Exchange）
3. 設定 Mantis 連線資訊
4. 設定備份排程

## 2. 啟動應用程式

### 從原始碼啟動

```bash
python -m hcp_cms
```

### 從執行檔啟動

雙擊 `HCP_CMS.exe` 或 `HCP_CMS.app`。

啟動後會看到主視窗，左側為導覽列，右側為內容區域。預設顯示儀表板。

## 3. 儀表板

儀表板提供本月客服工作的即時概覽。

### 3.1 KPI 卡片

| KPI | 說明 |
|-----|------|
| **本月案件** | 當月所有案件總數 |
| **回覆率** | 已回覆案件 ÷ 總案件數 × 100% |
| **待處理** | 狀態為「處理中」的案件數 |
| **平均 FRT** | 平均首次回覆時間（小時），排除超過 720 小時的異常值 |

### 3.2 最近案件

顯示當月最近 10 筆案件，包含案件編號、公司、主旨、狀態、時間。

### 3.3 重新整理

切換到儀表板頁面時會自動重新整理數據。

## 4. 案件管理

### 4.1 案件列表

- **篩選**：透過下拉選單篩選狀態（全部 / 處理中 / 已回覆 / 已完成 / Closed）
- **搜尋**：在搜尋框輸入關鍵字搜尋案件
- **重新整理**：點擊「🔄 重新整理」更新列表

### 4.2 案件詳情

點擊案件列表中的某一行，下方詳情面板會顯示：
- 案件編號
- 主旨
- 狀態
- 處理進度

### 4.3 案件操作

| 操作 | 說明 |
|------|------|
| **✅ 標記已回覆** | 將案件狀態改為「已回覆」，記錄回覆時間 |
| **🔒 結案** | 將案件狀態改為「已完成」 |
| **➕ 手動建案** | 手動建立新案件（適用於電話、現場等非信件來源） |

### 4.4 案件 ID 格式

`CS-YYYY-NNN`，例如 `CS-2026-001`。每年流水號自動重置。

### 4.5 案件狀態流轉

```
處理中 → 已回覆 → 已完成 / Closed
         ↑
         └── 客戶再次來信時自動重開
```

### 4.6 對話串追蹤

- 同一客戶、相同主旨（去除 RE:/FW: 前綴）的信件會自動關聯為同一對話串
- 子案件的 `linked_case_id` 指向根案件
- 根案件的 `reply_count` 自動累計

### 4.7 案件詳情對話框

雙擊案件列表中的案件，開啟 3 分頁詳情對話框：

| 分頁 | 內容 |
|------|------|
| **基本資訊** | 案件所有欄位，可直接編輯並儲存 |
| **Mantis 關聯** | 關聯的 Mantis 票務列表，可新增/移除關聯 |
| **操作日誌** | 案件的操作歷程記錄 |

### 4.8 批次刪除案件

1. 點擊案件管理頁面的「🗑️ 刪除案件」按鈕
2. 設定要刪除的日期範圍
3. 系統顯示符合條件的案件數量
4. 確認後執行刪除（刪除前建議先備份資料庫）

> **注意：** 刪除後無法復原。KMS 知識庫中已發布的條目不會被刪除。

### 4.9 CSV 匯入案件

透過 CSV 檔案批次匯入案件，適用於從其他系統遷移資料。

**匯入流程（3 步驟精靈）：**
1. **選擇檔案** — 選擇 CSV 檔案，系統自動偵測編碼
2. **欄位對應** — 將 CSV 欄位對應至案件欄位（含自定義欄位），設定衝突處理策略
3. **執行匯入** — 預覽匯入數量後確認執行，顯示匯入結果

### 4.10 自定義欄位

案件管理支援自定義欄位，可依需求新增額外的資料欄位。

1. 在案件管理頁面或 CSV 匯入精靈中建立新自定義欄位
2. 自定義欄位會自動出現在案件詳情對話框中
3. CSV 匯入時可將 CSV 欄位對應至自定義欄位

## 5. KMS 知識庫

### 5.1 搜尋

1. 在搜尋框輸入關鍵字（支援中文自然語言）
2. 點擊「🔍 搜尋」或按 Enter
3. 系統會自動：
   - 使用 jieba 進行中文斷詞
   - 查詢同義詞表擴展搜尋範圍（例如搜「薪水」也會找到「薪資」「工資」）
   - 以 BM25 相關度排序結果

### 5.2 搜尋結果

點擊搜尋結果，下方面板顯示：
- **問題**：問題描述
- **回覆**：回覆內容
- **解決方案**：具體解決步驟

### 5.3 新增 QA

點擊「➕ 新增 QA」手動建立知識條目。

### 5.4 QA 來源

| 來源 | 說明 |
|------|------|
| **email** | 信件處理時自動抽取對話串問答，儲存為「待審核」狀態，人工確認後才進入搜尋索引 |
| **manual** | 手動新增（直接為「已完成」狀態，立即可搜尋） |
| **import** | Excel 批次匯入 |

### 5.5 待審核 QA 審核

匯入含對話串的 .MSG 後，系統自動切割客戶問題段與我方回覆段，儲存為待審核 QA。

**審核流程：**
1. 點擊左側「📚 KMS 知識庫」
2. 切換至「**待審核**」分頁（有待審件時顯示 🔴N）
3. 點選 QA 條目，點擊「✏️ 編輯審核」開啟對話框
4. 確認或修改問題、回覆、解決方案、關鍵字、產品欄位
5. 按「**💾 儲存草稿**」暫存修改，條目仍保留在待審核（不進入搜尋索引）
6. 按「**✅ 確認完成**」正式核准，條目移至「全部」分頁並加入 FTS 搜尋索引
7. 不需要的條目可點「🗑️ 刪除」移除

> **注意：** 已完成的 QA 無法降回待審核狀態。匯出 Excel 時待審核 QA 不會出現在匯出檔案中。

### 5.6 批次匯入

KMS 知識庫支援多種批次匯入方式：

| 按鈕 | 說明 |
|------|------|
| **📂 批次匯入 .msg** | 多選 Outlook .msg 檔案，逐一解析對話串並建立待審核 QA |
| **📁 匯入整個資料夾** | 選擇資料夾後遞迴讀取所有 .msg / .xlsx / .csv 並批次匯入 |
| **📊 匯入 Excel/CSV** | 選擇單一 Excel 或 CSV 檔案匯入（支援欄位別名：問題/question/Q、回覆/answer/A 等） |

### 5.7 Excel 匯出

將已完成知識庫條目匯出為 Excel，供離線查閱或分享（待審核 QA 不含在內）。

### 5.8 智慧查詢

「🧠 智慧查詢」頁籤可將客戶來信原文貼入，系統自動比對 KMS 知識庫，找出最相似的前 10 筆解答。

**使用步驟：**
1. 切換至「🧠 智慧查詢」頁籤
2. 在輸入框貼入客戶問題原文（可整段貼入）
3. 點擊「🔍 智慧搜尋」
4. 從結果列表選擇最相符的條目
5. 下方顯示建議回覆內容，可直接複製使用

### 5.9 同義詞管理

在 **規則設定** 頁面管理同義詞群組，例如：
- 薪水 ↔ 薪資 ↔ 工資 ↔ 月薪
- 請假 ↔ 休假 ↔ 假單
- 離職 ↔ 辭職 ↔ 退休

## 6. 信件處理

### 6.1 信件來源設定

在下拉選單選擇信件來源：

| 來源 | 說明 |
|------|------|
| **IMAP** | 標準 IMAP 郵件伺服器（Gmail、自建伺服器等） |
| **Exchange** | Microsoft Exchange / Office 365（EWS 協定） |
| **.msg 手動匯入** | 從本機選擇 Outlook .msg 檔案匯入 |

### 6.2 操作模式

**全自動模式：**
1. 設定自動排程間隔（預設 15 分鐘）
2. 點擊「啟動自動處理」
3. 系統會定時抓取新信件並自動建案

**半自動模式：**
1. 設定日期範圍
2. 點擊「📥 取得信件列表」
3. 勾選要匯入的信件
4. 點擊「✅ 匯入勾選」

**手動模式：**
1. 點擊「📁 匯入 .msg 檔案」
2. 選擇一個或多個 .msg 檔案
3. 系統自動處理並建案

### 6.3 信件處理流程

每封信件經過 7 步處理管線：

1. **重複檢查** — SHA256 hash 防止重複建案
2. **廣播信過濾** — 系統公告不建案
3. **欄位解析** — 解析寄件人、主旨、本文、日期；草稿 .msg 自動從本文補回寄件人
4. **自動分類** — 依規則判斷產品、問題類型、錯誤類型、優先級、技術人員、處理進度
5. **對話串比對** — 辨識是否為已知案件的後續來信，自動關聯父案件
6. **建案** — 所有信件（含我方回覆）均建立案件；後續來信自動重開已回覆的父案件
7. **QA 抽取** — 偵測信件中嵌入的對話串（客戶問題段 + 我方回覆段），自動儲存為「待審核」KMS 條目，等待人工確認

### 6.4 對話串識別

系統自動識別 Outlook 引用格式（英文 `From:` 或中文 `寄件者:`）：
- **找到非我方（非 @ares.com.tw）的引用來源** → 上方為我方回覆段、下方為客戶問題段，自動抽取為待審核 QA
- **客戶再次來信（RE: 相同主旨）** → 已回覆案件自動重開為「處理中」，reply_count 自動累計

## 8. 報表中心

### 8.1 報表類型

| 頁籤 | 內容 |
|------|------|
| 客戶索引 | 客戶清單 + 案件數 + 快速連結至各客戶頁籤 |

|------|------|

### 8.2 產生報表

1. 選擇報表類型（追蹤表 / 月報）
2. 設定年份和月份
3. 點擊「📥 產生並下載」
4. 選擇儲存位置

### 8.3 報表排程

在系統設定中可設定自動產生報表的排程（例如每月 1 號自動產生上月月報），產生後自動建立郵件草稿。

### 8.4 Excel 樣式

- 字型：微軟正黑體 11pt

- 交替行：白色 / 淡灰色（#F9FAFB）
- 邊框：細線（#CCCCCC）

## 9. 規則設定

### 9.1 分類規則

系統依據正則表達式規則自動分類信件。規則存儲在資料庫中，可透過 GUI 管理。

| 規則類型 | 說明 | 範例 |
|---------|------|------|
| **product** | 系統產品 | `WebLogic\|WLS` → WebLogic |
| **issue** | 問題類型 | `bug\|錯誤\|異常` → BUG |
| **error** | 錯誤類型/功能模組 | `薪資\|薪水` → 薪資獎金計算 |
| **priority** | 優先等級 | `緊急\|urgent` → 高 |
| **broadcast** | 廣播信偵測 | `維護客戶\|更新公告` → 廣播 |
| **handler** | 自動指派技術人員 | `薪資\|薪酬` → 王小明 |
| **progress** | 自動填入處理進度 | `緊急\|ASAP` → 優先處理 |

### 9.2 管理規則

1. 選擇規則類型
2. 查看現有規則清單
3. 新增規則：
   - **Pattern**：正則表達式（如 `bug|錯誤|異常`）
   - **Value**：匹配值（如 `BUG`）
   - **Priority**：優先級（數字越小越優先，第一個匹配的規則生效）
4. 點擊「➕ 新增規則」

### 9.3 CSV 批次匯入/匯出

適合一次建立大量規則，或在多人環境中共用規則設定。

**匯出 CSV：**
1. 點擊「📤 匯出 CSV」
2. 選擇儲存位置
3. 產生 `rules.csv`，可用 Excel 編輯

**匯入 CSV：**
1. 準備 CSV 檔案，格式如下：
   ```csv
   rule_type,pattern,value,priority
   handler,薪資|薪酬,王小明,10
   handler,報表|報告,陳小華,20
   progress,緊急|ASAP,優先處理,5
   issue,bug|錯誤,BUG,10
   ```
2. 點擊「📥 匯入 CSV」
3. 選擇檔案後系統自動批次寫入，並顯示匯入結果（成功 N 筆、跳過 N 筆）

> **注意：** `priority` 欄位可省略，預設為 0。pattern 或 value 為空的列會自動跳過。

### 9.4 規則比對邏輯

- 比對文本 = 主旨 + 本文前 300 字
- 依 priority 排序，第一個匹配的規則生效
- 無匹配時使用預設值：產品=HCP、問題類型=OTH、錯誤類型=人事資料管理、優先級=中、技術人員=空白、處理進度=空白

## 11. 資料庫備份與還原

### 11.1 自動備份

- 系統依設定的頻率自動備份
- 備份儲存位置：
  - Windows：`%APPDATA%/HCP_CMS/backups/`
  - macOS：`~/Library/Application Support/HCP_CMS/backups/`
- 檔案命名：`cs_tracker_YYYYMMDD_HHMMSS.db`
- 超過保留天數的備份自動清理

### 11.2 手動備份/還原

**備份：**
1. 前往系統設定
2. 點擊「💾 立即備份」

**還原：**
1. 點擊「📦 還原」
2. 選擇備份檔案（.db）
3. 確認還原（還原前會自動備份當前資料庫）

### 11.3 匯出/匯入

**匯出（換電腦遷移用）：**
1. 點擊「📤 匯出」
2. 選擇儲存位置
3. 產生 .zip 檔案（含 DB + 設定）

**匯入合併（多人合併用）：**
1. 點擊「📥 匯入合併」
2. 選擇另一台電腦的 .db 檔案
3. 系統顯示預覽：新增 N 筆、衝突 N 筆
4. 選擇衝突處理策略：
   - **保留本機** — 衝突時略過匯入資料
   - **保留匯入** — 衝突時覆蓋本機資料
5. 確認後執行合併

## 12. 舊版系統遷移

如果你有舊版 HCP 客服工具包的 `cs_tracker.db`：

1. 前往系統設定
2. 點擊「🔄 舊 DB 遷移」

4. 系統自動偵測舊版 schema
5. 顯示預覽：可遷移 N 筆案件、N 筆 QA、N 筆 Mantis
6. 確認後執行遷移

- 遷移前自動備份當前資料庫

## 14. 常見問題（FAQ）

### Q: 資料庫被鎖定怎麼辦？

**A:** 當多人同時寫入 SQLite 時可能出現鎖定。系統已設定 5 秒等待 + 自動重試。如果仍然失敗，請等待其他使用者操作完成後重試。

### Q: 信件處理後沒有看到新案件？

**A:** 請點擊案件管理頁面的「🔄 重新整理」按鈕。

### Q: 搜尋 KMS 找不到結果？

**A:** 嘗試以下方法：
1. 使用不同的關鍵字
2. 在規則設定中新增同義詞
3. 確認 QA 條目已建立並被索引

### Q: 如何新增客戶公司？

**A:** 目前客戶公司在信件處理時自動從 sender email domain 識別。若需手動新增，可在資料庫中直接操作 companies 表。

### Q: 報表 Excel 無法開啟？

**A:** 確認：
1. 檔案沒有被其他程式開啟（關閉 Excel 後重試）
2. 磁碟空間充足
3. 目標路徑有寫入權限

### Q: 產生報表時顯示「產生失敗」？

**A:** 通常是案件的 progress 欄位含有 Email 原始文字夾帶的特殊控制字元。系統已內建自動過濾機制（v2.0.0 起），若仍出現問題請確認應用程式為最新版本。

### Q: 如何從 IMAP 改為 Exchange？

**A:** 在信件處理頁面的「來源」下拉選單切換即可。帳號密碼需在系統設定中重新設定。

### Q: 打包後的 exe 檔案很大？

**A:** PyInstaller 打包會包含 Python 執行環境和所有依賴，預計 150-250MB。這是正常的。

### Q: 如何升級系統？

**A:**
1. 備份資料庫
2. 下載新版本
3. 覆蓋舊檔案
4. 系統會自動偵測 schema 版本並執行必要的遷移
